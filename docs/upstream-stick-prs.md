# Upstream stick PRs surfaced by pigo dogfooding

While porting the `hashtagueule` Pico site (PHP-Twig 1.x theme) onto pigo, ten independent gaps in [`tyler-sommer/stick`](https://github.com/tyler-sommer/stick) were identified and patched. Seven (A–G) shipped in stick **v1.0.9**; two (H, I) are merged to `main` and waiting on a new tag; one (J) is still under maintainer review.

Kept as a record of the contribution and for the per-PR summaries.

| # | Branch | Status | Summary |
|---|---|---|---|
| A | `feature/include-array-ignore-missing` | v1.0.9 | `{% include [a, b, c] %}` array form + `ignore missing` modifier. Resolved the TODO at `parse/parse_tag.go:376`. New `IncludeNode.IgnoreMissing` field + `NewIncludeNodeWithOptions` constructor (`NewIncludeNode` retained as shim for back-compat). Executor iterates candidates via `env.Loader.Load`, picks the first that resolves; `ignore missing` swallows the load error. |
| B | `feature/endblock-name` | v1.0.9 | `{% endblock NAME %}` — accepts the optional closing block name and validates it matches the opening (Twig 3.x parity). Resolved the TODO at `parse/parse_tag.go:135`. |
| C | `feature/twig-builtin-tests` | v1.0.9 | New `twig/tests` package mirroring the `twig/filter` layout, registers Twig 3.x's built-in tests (`defined`, `divisible by`, `empty`, `even`, `iterable`, `null`/`none`, `odd`, `same as`) on `twig.New`. Includes a faithful `is defined` interceptor in `exec.go` that doesn't evaluate the left operand (so `{% set x = null %}; x is defined` correctly reports `true`). Plus a small parser fix to accept `null`/`none` keywords as test names after `is`. |
| D | `feature/twig-builtin-filters` | v1.0.9 | Implements six TODO'd filter stubs in `twig/filter/filter.go` to PHP-Twig 3.x semantics: `split`, `sort`, `slice`, `striptags`, `nl2br`, `number_format`. Stub-state was "return val unchanged", which broke any template using `\|split` (silent runtime errors) or any of the others (silent wrong output). |
| E | `feature/if-nil-else-no-panic` | v1.0.9 | Fixes a nil-pointer panic on `{% for x in xs if cond %}` when no item matches the inline `if`. Two-line fix in two places: `parseFor` constructs the per-iteration filter with an empty Else body (not nil), and the executor's IfNode walk gains a defensive `if node.Else == nil { return nil }` so any caller that builds an IfNode without an Else gets a clean no-op. |
| F | `feature/filter-merge-mixed` | v1.0.9 | `{}\|merge([x])` (hash + list) used to silently drop the list and return the empty hash. Restructure `filterMerge` so the hash-only path returns only when the right side is also a hash; otherwise fall through to the existing sequence-building path. Real-world impact: the tagblog theme's `{% set tbpages = tbpages\|merge([page]) %}` accumulator was a no-op every iteration, so `All Articles (0)` rendered instead of `All Articles (107)`. |
| G | `feature/spaceless-tag` | v1.0.9 | Implements the Twig 1.x `{% spaceless %}…{% endspaceless %}` tag (whitespace stripping between HTML tags via `>\s+<` → `><`, matching PHP `preg_replace`). Deprecated in Twig 2.x, removed in Twig 3.x in favor of the `\|spaceless` filter, but still common in older themes. Mirrors the existing `{% filter %}` shape exactly. |
| H | `feature/coerce-bool-collections` | merged, awaiting tag | `stick.CoerceBool` used to return `false` for any non-empty map/slice (the type switch fell through). Idioms like `{% if some_collection %}` silently skipped — themes had to rewrite as `{% if x\|length > 0 %}`. After scalar cases, reflect on `Kind()` and return `r.Len() > 0` for `Array/Slice/Map`. Nil slices/maps still → false. |
| I | `feature/twig-builtin-functions` | merged, awaiting tag | PHP-Twig `max` / `min` functions (the function form, not filters) were missing — `twig.New` built the env with an empty `Functions` map. Any template calling `max(counts)` errored with "Undeclared function". New `twig/fn` sub-package mirroring `twig/filter` / `twig/tests`; wired into `twig.New`. Docblock flags follow-ups (`range`, `cycle`, `date`, `dump`, `constant`, `attribute`, `random`, …) for the same package. |
| J | `feature/ordered-hash-literals` | PR open, under review | Hash literals (`{'a':1,'b':2}`) iterated in random order because stick evaluated them to `map[string]stick.Value` and walked them via `reflect.MapKeys()`. PHP-Twig guarantees insertion order. New `stick.Hash` ordered-map type, used for hash-literal eval and hash-returning filters; `Iterate`, `Len`, `IsMap`, `IsIterable`, `GetAttr`, `CoerceBool` get a `*Hash` case before reflect fallback. Maintainer flagged BC concern for user-defined extensions that type-assert `map[string]Value`; three mitigation paths proposed in the PR thread. |
